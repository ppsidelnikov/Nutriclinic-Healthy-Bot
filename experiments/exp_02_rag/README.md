# Эксперимент 2 (§3.2). Качество RAG-конфигураций по RAGAS

## Цель

Сравнить три конфигурации retrieval-пайплайна:
- **R1** — наивный плотный RAG (текущая реализация бота: top-k=3 dense)
- **R2** — гибридный поиск (dense + BM25 через RRF)
- **R3** — гибридный + cross-encoder reranking (top-N=10 → top-k=3)

Метрики — RAGAS: `faithfulness`, `answer_relevancy`, `context_precision`, `context_recall`.

Дополнительные подэксперименты (опционально):
- Размер чанка: 256 / 512 / 1024 токенов
- Модель эмбеддингов: text-embedding-3-small / BGE-M3 / multilingual-E5-large

## Что от тебя требуется

Заполнить файл `qa_set/qa_pairs.jsonl` — вручную составленные пары «вопрос — эталонный ответ» по нутрициологии. Минимум 30 пар, целевое число 50.

Формат каждой строки (JSONL):

```json
{"question": "<вопрос пользователя>", "ground_truth": "<эталонный ответ 1-3 предложения>", "reference_pdf": "<имя PDF из docs/, опционально>"}
```

Шаблон с примерами уже создан: `qa_set/qa_pairs.jsonl`.

### Рекомендации по составлению

1. **Покрывай реальные сценарии**: вопросы по конкретным продуктам, замены, диеты для целей (похудение / набор / поддержание), вопросы о КБЖУ, ограничения (диабет, аллергии), типичные заблуждения.
2. **Эталонный ответ пиши на основе материалов из `docs/`** — открой PDF, найди релевантный кусок, перепиши лаконично.
3. **Не более 3 предложений** в эталоне: длинные ответы хуже работают как ground truth.
4. **Указывай `reference_pdf`** — это позволит RAGAS точнее посчитать `context_recall`.
5. **Разнообразие важнее количества**: 30 разноплановых пар лучше 50 однотипных.

### Категории вопросов (для самопроверки полноты)

- [ ] Конкретный продукт (можно ли / как часто / при каком состоянии)
- [ ] Замены продуктов (чем заменить X)
- [ ] Расчёт нормы (сколько белка / калорий / воды нужно)
- [ ] Цели (как набрать массу / похудеть / поддерживать вес)
- [ ] Типичные заблуждения (правда ли что X)
- [ ] Ограничения по состоянию здоровья (диабет / гастрит / аллергия)
- [ ] Питание вокруг тренировок
- [ ] Анализ рациона (что не так / чего не хватает)

## Что делаю я (после получения QA-сета)

1. `pipelines/r1_naive_dense.py`, `r2_hybrid.py`, `r3_hybrid_rerank.py` — три конфигурации retrieval.
2. `run.py` — прогоняет QA-сет через все три конфигурации, сохраняет ответы и извлечённые контексты в `results/`.
3. `evaluate_ragas.py` — считает 4 метрики RAGAS на каждой конфигурации.
4. `analysis.ipynb` — таблицы и графики для диссертации.

## Запуск

```bash
cd experiments/exp_02_rag
python run.py --config R1 --qa qa_set/qa_pairs.jsonl --out results/r1_runs.jsonl
python run.py --config R2 --qa qa_set/qa_pairs.jsonl --out results/r2_runs.jsonl
python run.py --config R3 --qa qa_set/qa_pairs.jsonl --out results/r3_runs.jsonl
python evaluate_ragas.py --runs results/ --out results/metrics.csv
```
